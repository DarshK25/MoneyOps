## 1. All Bugs Fixed — Root Cause Summary

Every issue identified in the v1 schema analysis is listed below with its root cause, impact, and the exact structural change made in v2.

| Bug # | Issue | Root Cause | Impact | Fix in v2 |
|-------|-------|------------|--------|-----------|
| 1 | Invoice-to-Client join returned empty | `clients.id` was String but `invoices.clientId` was UUID | BSON type mismatch prevented $lookup aggregation | Both are now String |
| 2 | Payment queries returned empty arrays | `transactions.invoiceId` was UUID but `invoices.id` was String | Cross-reference broken at BSON level | Both are now String |
| 3 | Full collection scans on revenue metrics | Missing composite index on `{orgId, type}` | 2.3s query time on 10K records | Added compound index |
| 4 | Duplicate invoices from voice retry | No idempotency key enforcement | Voice agent retried on timeout, created duplicate invoices | Added `idempotencyKey` with unique index |
| 5 | Session data lost on gateway restart | Session stored in-memory dict only | Voice context lost mid-conversation | Added Redis + MongoDB fallback |
| 6 | No tenant isolation in queries | orgId was optional in service layer | Data leaked across organizations | Made orgId mandatory in all queries |
| 7 | Hard deletes destroyed audit trail | DELETE FROM directly removed records | Compliance violation, no recovery | Switched to soft delete with `deletedAt` |
| 8 | No audit trail for changes | Missing audit fields | Couldn't track who created/modified data | Added createdBy, updatedBy, timestamps |

## 2. Global Schema Rules (v2)

### 2.1 ID Strategy — All Collections

**Rule:** Every `@Id` field across ALL collections is `String`, storing a UUID in plain hyphenated text format (e.g. `'9f3b96db-7e13-489d-a5a4-18ecdbc3d616'`). No `java.util.UUID` type. No `BinData(3)`. No mixed representations.

**application.yml change required:**

```yaml
spring:
  data:
    mongodb:
      uuid-representation: STANDARD
```

**Java Entity:**

```java
@Id
private String id;  // NOT UUID

@PrePersist
public void generateId() {
    if (this.id == null) {
        this.id = UUID.randomUUID().toString();
    }
}
```

---

### 2.2 Tenant Isolation Rule

**Rule:** Every repository query MUST include `orgId` as a mandatory filter. Every controller endpoint MUST declare `@RequestHeader(value='X-Org-Id', required=true) String orgId`. A missing or blank header returns HTTP 403 immediately — no fallback, no default.

**Controller Example:**

```java
@GetMapping("/invoices")
public ResponseEntity<List<Invoice>> getInvoices(
    @RequestHeader(value = "X-Org-Id", required = true) String orgId
) {
    return ResponseEntity.ok(invoiceService.findByOrgId(orgId));
}
```

**Repository Query:**

```java
List<Invoice> findByOrgIdAndDeletedAtIsNull(String orgId);
```

---

### 2.3 Soft Delete

**Rule:** No collection uses hard deletes. Every collection gains a `deletedAt: LocalDateTime` field (`null` = active). All queries add a filter: `{ deletedAt: null }`. This preserves audit trails and prevents accidental data loss.

**Entity Field:**

```java
private LocalDateTime deletedAt;
```

**Soft Delete Method:**

```java
public void softDelete() {
    this.deletedAt = LocalDateTime.now();
}
```

**Query Filter:**

```java
List<Invoice> findByOrgIdAndDeletedAtIsNull(String orgId);
```

---

### 2.4 Audit Fields

**Rule:** Every collection gains:
- `createdAt` (LocalDateTime)
- `updatedAt` (LocalDateTime)
- `createdBy` (String — Clerk userId)
- `updatedBy` (String)

These are populated automatically by a Spring Data `@CreatedDate` / `@LastModifiedDate` AuditorAware bean.

**Entity Fields:**

```java
@CreatedDate
private LocalDateTime createdAt;

@LastModifiedDate
private LocalDateTime updatedAt;

@CreatedBy
private String createdBy;  // Clerk userId

@LastModifiedBy
private String updatedBy;
```

---

### 2.5 Idempotency Keys

**Rule:** Every write operation from the AI Gateway (create invoice, record payment, create client) must include an `idempotencyKey` (UUID generated by the gateway). The backend enforces uniqueness on this key per org, preventing duplicate writes from voice retry loops.

**Entity Field:**

```java
@Indexed(unique = true)
private String idempotencyKey;  // From AI Gateway
```

**Index:**

```javascript
db.invoices.createIndex(
    { "orgId": 1, "idempotencyKey": 1 },
    { unique: true, partialFilterExpression: { idempotencyKey: { $exists: true } } }
)
```

```javascript
// Tenant isolation
db.users.createIndex({ "orgId": 1, "deletedAt": 1 })

db.users.createIndex({ "orgId": 1, "role": 1, "deletedAt": 1 })
```

**Schema:**

```javascript
{
```

```javascript
  orgId: String,                  // 🔗 Tenant isolation
  name: String,
  email: String,
  phoneNumber: String,
  gstin: String,                  // Client's GST number (if applicable)
  billingAddress: {
    line1: String,
    line2: String,
    city: String,
    state: String,
    country: String,
    pincode: String
  },
  shippingAddress: {              // Optional, defaults to billing if null
    line1: String,
    line2: String,
    city: String,
    state: String,
    country: String,
    pincode: String
  },
  paymentTerms: Number,           // Days (default from org settings)
  metadata: {                     // ✨ Custom fields for voice context
    lastInvoiceDate: ISODate,
    totalInvoiced: Number,
    totalPaid: Number,
    outstandingBalance: Number
  },
  createdAt: ISODate,
  updatedAt: ISODate,
  createdBy: String,
  updatedBy: String,
  deletedAt: ISODate
}
```

**Indexes:**

```javascript
// Tenant isolation + soft delete
db.clients.createIndex({ "orgId": 1, "deletedAt": 1 })

// Search by name (case-insensitive)
db.clients.createIndex({ "orgId": 1, "name": 1, "deletedAt": 1 })

// Email lookup
db.clients.createIndex({ "orgId": 1, "email": 1, "deletedAt": 1 })

db.clients.createIndex({ "orgId": 1, "phoneNumber": 1, "deletedAt": 1 })
```

### 3.4 Collection: `invoices`

**⚠️ Critical Fix:** `invoices.clientId` was typed as UUID in Java but `clients.id` is String. The cross-reference was broken at the BSON level — payment lookups returned empty arrays even when payments existed. Both are now String.

**Schema:**

```javascript
{
  _id: String,                    // 🔑 UUID as string
  orgId: String,                  // 🔗 Tenant isolation
  clientId: String,               // 🔗 Reference to clients (⚠️ WAS UUID, NOW STRING)
  invoiceNumber: String,          // e.g., "INV-2024-001"
  status: String,                 // DRAFT | SENT | PAID | OVERDUE | CANCELLED
  issueDate: ISODate,
  dueDate: ISODate,
  lineItems: [                    // 📋 Embedded array
    {
      description: String,
      quantity: Number,
      unitPrice: Number,
      taxRate: Number,            // Percentage (e.g., 18 for 18% GST)
      taxAmount: Number,          // Calculated
      lineTotal: Number           // quantity * unitPrice + taxAmount
    }
  ],
  subtotal: Number,               // Sum of all lineItems.lineTotal (excluding tax)
  taxTotal: Number,               // Sum of all lineItems.taxAmount
  total: Number,                  // subtotal + taxTotal
  amountPaid: Number,             // ✨ Denormalized from transactions
  balanceDue: Number,             // total - amountPaid
  currency: String,               // INR | USD | EUR
  notes: String,                  // Internal notes
  termsAndConditions: String,
  pdfUrl: String,                 // S3/CloudFlare R2 URL
  voiceContext: {                 // ✨ Metadata from voice session
    sessionId: String,
    createdViaVoice: Boolean,
    transcript: String            // Original voice command
  },
  idempotencyKey: String,         // ✨ From AI Gateway (prevents duplicate creation)
  createdAt: ISODate,
  updatedAt: ISODate,
  createdBy: String,
  updatedBy: String,
  deletedAt: ISODate
}
```

**Indexes:**

```javascript
// Tenant isolation + soft delete
db.invoices.createIndex({ "orgId": 1, "deletedAt": 1 })

// Invoice number lookup (must be unique per org)
db.invoices.createIndex(
  { "orgId": 1, "invoiceNumber": 1 },
  { unique: true }
)

// Client invoices
db.invoices.createIndex({ "orgId": 1, "clientId": 1, "deletedAt": 1 })

// Status-based queries (e.g., "show all overdue invoices")
db.invoices.createIndex({ "orgId": 1, "status": 1, "deletedAt": 1 })

// Due date queries (for payment reminders)
db.invoices.createIndex({ "orgId": 1, "dueDate": 1, "status": 1, "deletedAt": 1 })

// Idempotency enforcement (⚠️ CRITICAL)
db.invoices.createIndex(
  { "orgId": 1, "idempotencyKey": 1 },
  {
    unique: true,
    partialFilterExpression: { idempotencyKey: { $exists: true, $ne: null } }
  }
)
```

### 3.5 Collection: `transactions`

**Bug Fix:** Missing index on `{orgId, type}` caused full collection scans for every revenue/expense metric computation. Idempotency key added to prevent double-payment from voice retry loops.

**Schema:**

```javascript
{
  _id: String,                    // 🔑 UUID as string
  orgId: String,                  // 🔗 Tenant isolation
  invoiceId: String,              // 🔗 Reference to invoices (⚠️ WAS UUID, NOW STRING)

  amount: Number,
  currency: String,

    sessionId: String,

  idempotencyKey: String,         // ✨ From AI Gateway (prevents double-payment)
  createdAt: ISODate,
  updatedAt: ISODate,
  createdBy: String,
  updatedBy: String,
  deletedAt: ISODate
}
```

**Indexes:**

```javascript

// Invoice payments (⚠️ CRITICAL - was missing, caused 2.3s query time)
```

## 4. New Collections (v2 Additions)

### 4.1 Collection: `voice_sessions` (Redis + Mongo fallback)

**✨ New:** Formalizes what was previously an unstructured in-memory dict. Redis is primary store with 3600s TTL. MongoDB is fallback when Redis is unavailable. This survives gateway restarts.

**Schema:**

```javascript
{
  _id: String,                    // 🔑 Session ID (UUID)
  orgId: String,                  // 🔗 Tenant isolation
  userId: String,                 // 🔗 Reference to users
  status: String,                 // ACTIVE | COMPLETED | ABANDONED | ERROR
  conversationHistory: [          // 📋 Array of turns
    {
      role: String,               // USER | ASSISTANT
      timestamp: ISODate,
      text: String,
      intent: String,             // Detected intent (CREATE_INVOICE, etc.)
      extractedEntities: Object   // NER results
    }
  ],
  draftInvoice: {                 // ✨ Work-in-progress invoice data
    clientName: String,
    amount: Number,
    description: String,
    dueDate: ISODate,
    confirmed: Boolean
  },
  metadata: {
    livekitRoomId: String,
    deviceType: String,           // WEB | MOBILE | WHATSAPP
    language: String              // en | hi
  },
  startedAt: ISODate,
  lastActivityAt: ISODate,
  expiresAt: ISODate,             // ✨ TTL for Redis (3600s from last activity)
  createdAt: ISODate,
  updatedAt: ISODate
}
```

**Redis key structure:**

```
voice:session:{sessionId} -> JSON string of above schema
TTL: 3600 seconds (auto-refresh on activity)
```

**Indexes (MongoDB fallback):**

```javascript
// Tenant isolation
db.voice_sessions.createIndex({ "orgId": 1 })

// User sessions
db.voice_sessions.createIndex({ "orgId": 1, "userId": 1, "status": 1 })

// Active session lookup
db.voice_sessions.createIndex({ "status": 1, "lastActivityAt": -1 })

// TTL index (auto-delete expired sessions after 24 hours)
db.voice_sessions.createIndex(
  { "expiresAt": 1 },
  { expireAfterSeconds: 0 }
)
```

### 4.2 Collection: `idempotency_keys`

**✨ New:** Dedicated idempotency tracking collection. Catches duplicate requests at the gateway level before they reach the domain logic, preventing double-invoices and double-payments from voice retry loops.

**Schema:**

```javascript
{
  _id: String,                    // 🔑 The idempotency key itself (UUID from AI Gateway)
  orgId: String,                  // 🔗 Tenant isolation
  operation: String,              // CREATE_INVOICE | RECORD_PAYMENT | CREATE_CLIENT
  entityType: String,             // INVOICE | TRANSACTION | CLIENT
  entityId: String,               // ID of the created entity (for deduplication response)
  requestPayload: Object,         // Original request (for debugging)
  responsePayload: Object,        // Original response (returned on duplicate)
  createdAt: ISODate,
  expiresAt: ISODate              // TTL: 7 days (voice session can retry for up to 1 day)
}
```

**Indexes:**

```javascript
// Primary lookup (⚠️ CRITICAL - enforces uniqueness)
db.idempotency_keys.createIndex(
  { "_id": 1, "orgId": 1 },
  { unique: true }
)

// Cleanup old keys
db.idempotency_keys.createIndex(
  { "expiresAt": 1 },
  { expireAfterSeconds: 0 }
)

// Entity lookup (for debugging)
db.idempotency_keys.createIndex({ "orgId": 1, "entityType": 1, "entityId": 1 })
```

## 5. Complete Relationship Map

```
business_organizations (1)
    ↓
    └─→ users (N)
        ├─→ createdBy (in all collections)
        └─→ updatedBy (in all collections)
    
    └─→ clients (N)
        ├─→ invoices (N)
        │   ├─→ lineItems (embedded array)
        │   ├─→ transactions (N)
        │   └─→ documents (N)
        │
        └─→ transactions (N)
            └─→ documents (N)
    
    └─→ voice_sessions (N)
        └─→ conversationHistory (embedded array)
    
    └─→ audit_logs (N)
        └─→ entityId (polymorphic reference)
    
    └─→ idempotency_keys (N)
        └─→ entityId (polymorphic reference)
    
    └─→ documents (N)
        └─→ entityId (polymorphic reference)
```

**Key Constraints:**

1. **Tenant Isolation:** Every query MUST filter by `orgId`
2. **Soft Delete:** Every query MUST filter by `deletedAt: null`
3. **Type Consistency:** All IDs are `String` (UUID format), no mixed types
4. **Idempotency:** All write operations from AI Gateway include `idempotencyKey`

## 6. MongoDB Atlas Migration Script

Run in order. Each block is idempotent — safe to re-run. Run in Atlas console or via `mongosh`.

### 6.1 — UUID Representation (application.yml)

**⚠️ CRITICAL:** Change in your Spring Boot `application.yml` BEFORE running any migration. This changes how Java serializes UUIDs to MongoDB.

```yaml
spring:
  data:
    mongodb:
      uuid-representation: STANDARD  # Changed from UNSPECIFIED
```

## 7. Java Entity Changes Required

These are the exact changes needed in each Spring Boot entity class. All other logic (services, controllers) flows from these.

### 7.1 — Every Entity: Change UUID → String

**Before:**

```java
@Id
private UUID id;
```

**After:**

```java
@Id
private String id;

@PrePersist
public void generateId() {
    if (this.id == null) {
        this.id = UUID.randomUUID().toString();
    }
}
```

### 7.2 — Controller: Enforce X-Org-Id Header

**Before:**

```java
@GetMapping("/invoices")
public ResponseEntity<List<Invoice>> getInvoices() {
    return ResponseEntity.ok(invoiceService.findAll());
}
```

**After:**

```java
@GetMapping("/invoices")
public ResponseEntity<List<Invoice>> getInvoices(
    @RequestHeader(value = "X-Org-Id", required = true) String orgId
) {
    if (orgId == null || orgId.isBlank()) {
        return ResponseEntity.status(HttpStatus.FORBIDDEN).build();
    }
    return ResponseEntity.ok(invoiceService.findByOrgId(orgId));
}
```

### 7.3 — Session Manager: Add TTL and draft_expires_at

**VoiceSession Entity:**

```java
@Document(collection = "voice_sessions")
public class VoiceSession {
    
    @Id
    private String id;
    
    private String orgId;
    private String userId;
    private String status;
    
    private List<ConversationTurn> conversationHistory;
    private DraftInvoice draftInvoice;
    
    private LocalDateTime startedAt;
    private LocalDateTime lastActivityAt;
    
    @Indexed(expireAfterSeconds = 0)  // TTL index
    private LocalDateTime expiresAt;  // ✨ NEW: Auto-delete after expiry
    
    private LocalDateTime createdAt;
    private LocalDateTime updatedAt;
    
    @PrePersist
    public void onCreate() {
        if (this.id == null) {
            this.id = UUID.randomUUID().toString();
        }
        this.startedAt = LocalDateTime.now();
        this.lastActivityAt = LocalDateTime.now();
        this.expiresAt = LocalDateTime.now().plusHours(1);  // 1 hour TTL
        this.createdAt = LocalDateTime.now();
        this.updatedAt = LocalDateTime.now();
    }
    
    @PreUpdate
    public void onUpdate() {
        this.lastActivityAt = LocalDateTime.now();
        this.expiresAt = LocalDateTime.now().plusHours(1);  // Extend TTL on activity
        this.updatedAt = LocalDateTime.now();
    }
}
```

## 9. Troubleshooting Guide

### Issue: Invoice-to-Client Join Returns Empty

**Symptom:** `invoices.clientId` exists but lookup returns no client

**Root Cause:** Type mismatch between `invoices.clientId` and `clients._id`

**Fix:**

```javascript
// Check types in MongoDB Compass
db.invoices.findOne({ clientId: { $exists: true } })
// If clientId is BinData(3), run conversion:
db.invoices.find({ clientId: { $type: "binData" } }).forEach(function(doc) {
  db.invoices.updateOne(
    { _id: doc._id },
    { $set: { clientId: doc.clientId.toString() } }
  );
});
```

### Issue: Duplicate Invoices Created from Voice

**Symptom:** Same voice command creates 2+ invoices

### Issue: Session Lost After Gateway Restart

**Symptom:** User loses voice context mid-conversation after deployment

**Root Cause:** Session stored only in-memory, not in Redis/MongoDB

**Fix:**

```python
# AI Gateway: Use Redis for session storage
import redis

redis_client = redis.Redis(host='localhost', port=6379, db=0)

def save_session(session_id, session_data):
    redis_client.setex(
        f"voice:session:{session_id}",
        3600,  # 1 hour TTL
        json.dumps(session_data)
    )

def get_session(session_id):
    data = redis_client.get(f"voice:session:{session_id}")
    if data:
        return json.loads(data)
    
    # Fallback to MongoDB
    return mongo_db.voice_sessions.find_one({"_id": session_id})
```
